## ⚡ Step-by-Step Setup

### Step 1 — Create the Profile Repository
1. Go to `github.com` → Click **New Repository**
2. Name it **exactly** `VivekShukla-2504` (same as your username)
3. Check ✅ **Public**
4. Check ✅ **Add a README file**
5. Click **Create Repository**

---

### Step 2 — Upload Files
1. Copy the `README.md` content into your repo's README
2. Create the `assets/` folder and upload:
   - `header-banner.svg`
   - `skills-chart.svg`
3. Create `.github/workflows/` folder and upload:
   - `snake.yml`
   - `metrics.yml`
   - `update-readme.yml`

---

### Step 3 — Enable GitHub Actions (Snake Animation)
1. Go to your repo → **Settings** → **Actions** → **General**
2. Under **Workflow permissions**, select **Read and write permissions**
3. Click **Save**
4. Go to **Actions** tab → Click **Generate Snake Animation**
5. Click **Run workflow** → **Run workflow** ✅
6. Wait 1-2 minutes → Check the `output` branch for the SVG files

---

### Step 4 — Set Up Secrets (For Metrics)

Go to repo → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

| Secret Name | Value | How to Get |
|-------------|-------|-----------|
| `METRICS_TOKEN` | GitHub Personal Access Token | [Create PAT](https://github.com/settings/tokens) with `public_repo`, `read:user` scopes |
| `WAKATIME_TOKEN` | WakaTime API Key | [WakaTime Settings](https://wakatime.com/settings/account) |

---

### Step 5 — Enable Actions for Metrics
1. Go to **Actions** tab
2. Click **Update GitHub Metrics**
3. Click **Run workflow** → **Run workflow**
4. Metrics SVGs will appear in `metrics/` folder after ~5 min

---

### Step 6 — Optional Enhancements

#### Add WakaTime Coding Stats
1. Install [WakaTime extension](https://wakatime.com/plugins) in VS Code
2. Sign up at [wakatime.com](https://wakatime.com)
3. Add `WAKATIME_TOKEN` secret (see Step 4)

#### Add GitHub Metrics (Advanced)
1. Create a [fine-grained PAT](https://github.com/settings/tokens?type=beta)
2. Required scopes: `Contents (Write)`, `Metadata (Read)`, `Actions (Read)`
3. Add as `METRICS_TOKEN` secret
